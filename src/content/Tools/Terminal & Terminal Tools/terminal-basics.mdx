## üß† 1. What is the Terminal?

Your computer has two interfaces: the one with windows and icons you click, and the terminal - a text box where you type commands directly. Developers use it because it's faster, more powerful, and because most dev tools (like Git and Node) are designed for it first.

The terms **terminal**, **shell**, and **command line** are often used interchangeably. Technically they're different things, but for our purposes they all mean the same thing: the place where you type commands.

## üíø 2. Opening a Terminal

### üçé macOS / üêß Linux

Your machine comes with a perfectly good terminal out of the box.

**Built-in (works great):**
Open **Spotlight** (`Cmd + Space`), type "Terminal", and hit Enter.

**Recommended upgrade:**
[iTerm2](https://iterm2.com/) is a free replacement with better tab support, split panes, and search. Install it via Homebrew:
```bash
brew install --cask iterm2
```

Either way, the commands are identical. It's just a window preference.

### ü™ü Windows

Windows has two older terminals (`cmd.exe` and PowerShell) but neither uses the same commands as macOS/Linux. The fix is **Windows Subsystem for Linux (WSL)**, which gives you a real Linux terminal on Windows. This is what the rest of this guide assumes.

**Install WSL:**
Open PowerShell as Administrator and run:
```powershell
wsl --install
```

Restart your machine when prompted. After rebooting, search for **Ubuntu** in the Start Menu. That's your new terminal.

> **Why WSL?** Every dev tool, tutorial, and guide (including ours) is written with Mac/Linux commands in mind. WSL means you can follow along without translating every command.

## ‚ö° 4. Essential Commands

These are the commands you will use constantly. The syntax is the same on macOS, Linux, and WSL.

### Navigating Around

Think of the terminal as a file explorer where you type instead of click. At any moment, you are "inside" a folder. This is called your <Tooltip tip="The folder you are currently inside in the terminal. Commands run relative to this location.">working directory</Tooltip>.

**Where am I?**
```bash
pwd
```
_Prints the full path of your current folder, e.g. `/Users/yourname/projects`._

**What's in here?**
```bash
ls
```
_Lists all files and folders in the current directory._
```bash
ls -la
```
_Same, but also shows hidden files (ones starting with `.`) and file permissions._

**Move into a folder:**
```bash
cd projects
```

**Move up one level:**
```bash
cd ..
```

**Go straight home:**
```bash
cd ~
```
_`~` is a shortcut for your home directory (`/Users/yourname` on Mac, `/home/yourname` on Linux/WSL)._

**Jump anywhere with a full path:**
```bash
cd ~/projects/my-app
```

### Managing Files & Folders

**Create a new folder:**
```bash
mkdir my-project
```

**Create a new file:**
```bash
touch index.html
```

**Copy a file:**
```bash
cp index.html index-backup.html
```

**Move or rename a file:**
```bash
mv index.html src/index.html   # move
mv old-name.html new-name.html # rename
```

**Delete a file:**
```bash
rm index.html
```

**Delete a folder and everything inside it:**
```bash
rm -rf my-project
```

> ‚ö†Ô∏è `rm -rf` is permanent. There's no Recycle Bin. Double-check your path before running it.

---

### Reading Files

**Print a file's contents:**
```bash
cat package.json
```

**Open a file in VS Code from the terminal:**
```bash
code index.html
```

**Open your entire current folder in VS Code:**
```bash
code .
```

## ü™ü Windows-Specific Notes

If you're ever stuck in plain PowerShell or `cmd.exe` instead of WSL, here's the translation for the most common commands:

| What you want | macOS / Linux / WSL | PowerShell         |
|:--------------|:--------------------|:-------------------|
| Where am I?   | `pwd`               | `pwd`              |
| List files    | `ls`                | `dir`              |
| Change folder | `cd projects`       | `cd projects`      |
| Create folder | `mkdir my-project`  | `mkdir my-project` |
| Create file   | `touch index.html`  | `ni index.html`    |
| Delete file   | `rm file.html`      | `rm file.html`     |
| Delete folder | `rm -rf folder`     | `rm -r folder`     |

That being said... just use WSL. Despite the similarities in some commands above, UNIX is superior in almost every way.

## üêõ Troubleshooting

### "command not found"

The terminal doesn't recognize what you typed.

- **Fix:** Check for typos. If the command is something you just installed (like `git` or `node`), try closing and reopening your terminal, as it needs to reload to pick up new programs.

### "permission denied"

You don't have the rights to run that command or access that file.

- **Fix:** On macOS/Linux, prefix the command with `sudo` to run it as an administrator: `sudo rm locked-file.txt`. It will ask for your password.

### My terminal opened in the wrong folder

- **Fix:** Use `cd` to navigate where you need to go, or open VS Code in the right folder first and use the integrated terminal. It will open directly in your project root.

### WSL can't find my Windows files

Your Windows files live at `/mnt/c/` in WSL. So `C:\Users\yourname\projects` becomes:
```bash
cd /mnt/c/Users/yourname/projects
```
